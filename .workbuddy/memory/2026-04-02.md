## 修改：智谱 GLM quota/limit 解析逻辑

**文件**：`src-tauri/src/providers/zhipu.rs`

**问题**：原代码通过 `nextResetTime` 升序排序来区分5小时窗口和每周额度，不够准确。

- `unit=3`（小时），`number=5` → 每5小时滑动窗口（TOKENS_LIMIT）
- `unit=6`（周），`number=1` → 每周 Token 总额度（TOKENS_LIMIT）
- `TIME_LIMIT` → MCP 月度调用次数

**修改内容**：
1. 将 TOKENS_LIMIT 区分策略改为：直接匹配 `unit=3` → `window_quota`，`unit=6` → `week_quota`
2. 其他 unit 值保留旧的 nextResetTime 排序兜底逻辑
3. `build_label` 函数重构：按 unit 类型（3=小时,5=天,6=周,7=月）+number 数量生成标签
4. 更新顶部注释说明 unit 字段语义
